1. Multi-Page Application
Your app must be built as a multi-page web application using the tools and techniques covered in class.

2. User Authentication
Your app must include a working:

Registration system

Login system

Logout functionality

3. Two Distinct User Roles
Your app must support at least two different user roles with different permissions, workflows, or dashboards.

Examples:

Customer vs. Store Owner

Patient vs. Doctor

Your system must correctly identify users after login and route them to the appropriate interface.

4. Meaningful CRUD Functionality
Your application must include meaningful Create, Read, Update, and Delete operations across the system.

Important clarification: this does not mean both roles must have identical CRUD privileges. Permissions should make sense for the app. For example, one role may create and edit records, while another role may only view, update, or request changes depending on the workflow.
